This application has been implemented in the following scenario using ocLazyLoad and requireJs.

  • On the landing page, , we have attached landing controllers to the main app. We are attaching the controllers and services lazily, we used registerController, $register.factory functionality which is defined in the app.config. When the files are loaded lazily the controller and the services will be attached to the main app.
  • For home controller, we have used a separate module called "SampleApp.Home". Therefor after loading the file, we need to inject the module using ocLazyLoad. This functionality is implemented in the app.js file at the routing "/home" (see resolve block in code).
  • To load page1, we have used the module "SampleApp.Pages". Page1Service.js file has a dependency to page1Controller.js file. This dependent file has to loaded before loading the page1Controller.js file.
    We created the module "SampleApp.Pages" in Page1Service.js file, Since Page1Service.Js file is a parent in the hierarchy.
  • We created a separate routing in "SampleApp.Pages".
  • We created the controller and services under "SampleApp.Pages" for page2. This page will be loaded lazily after the first module has been loaded. We therefore need to register the controllers and services for this.
